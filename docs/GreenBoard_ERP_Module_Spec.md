# 그린보드 일렉트로닉스 ERP 모듈 상세 설계서

## 1. 시스템 개요

| 항목 | 내용 |
|------|------|
| **시스템명** | GreenBoard ERP (Enterprise Resource Planning) |
| **대상 기업** | (주)그린보드 일렉트로닉스 |
| **업종** | PCB/전자부품 제조 |
| **규모** | 중견기업 (직원 350명, 연매출 1,800억원) |

## 8. 개발 우선순위

### Phase 1: 핵심 기능 (24개 화면)
- 기준정보: 공통코드, 품목, 거래처, BOM (5개)
- 구매관리: 발주, 입고 (4개)
- 영업관리: 수주, 출하 (4개)
- 재고관리: 재고현황, 입출고 (3개)
- 생산관리: 제조오더, MRP (4개)
- 시스템: 사용자, 권한 (4개)

### Phase 2: 확장 기능 (20개 화면)
- 영업관리: 견적, 매출, 채권 (4개)
- 구매관리: 매입, 업체평가 (3개)
- 회계관리: 전표, 장부 (4개)
- 인사급여: 사원, 근태, 급여 (5개)
- 재고관리: 안전재고, 장기재고 (2개)
- 생산관리: MPS, 실적분석 (2개)

### Phase 3: 고도화 (12개 화면)
- 나머지 전체 화면
- AI 연계 강화
- 리포트/대시보드
